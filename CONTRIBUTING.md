Most content in this repository is **automatically generated** by CI/CD pipelines from private app repositories. The following directories are auto-updated:

- `*/changelog/` - Generated by GitHub Actions on each release
- `*/changelog/entries/` - Individual release entries
- `*/changelog/index.json` - Full changelog index
- `*/changelog/latest.json` - Latest version metadata

**Please do not manually edit these files**, as they will be overwritten on the next automated update.

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b docs/improve-readme`
3. **Make your changes**
4. **Test if applicable** (e.g., validate JSON against schemas)
5. **Commit with descriptive message**: `docs: improve changelog access examples`
6. **Push to your fork**: `git push origin docs/improve-readme`
7. **Open a Pull Request**
